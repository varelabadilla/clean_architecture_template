# Infrastructure / Persistence

## Why This Folder Exists

This folder contains everything related to **data persistence**: the database context, entity configuration, migrations, and repository implementations. It is the bridge between the domain model (pure objects with no ORM knowledge) and the relational or document database that stores them.

Isolating all persistence concerns here means the Domain layer remains completely free of ORM annotations, table mapping logic, and database-specific types.

---

## What Belongs Here

- `AppDbContext.cs` — the EF Core database context (or JPA EntityManager configuration)
- `Configurations/` — per-entity fluent API configurations
- `Migrations/` — auto-generated EF Core database migrations
- `Repositories/` — concrete implementations of Domain repository interfaces

---

## What Does NOT Belong Here

- Business logic
- Domain entities (those live in `Domain/Entities/`)
- Repository interfaces (those live in `Domain/Repositories/`)
- Raw SQL scripts not generated by migrations

## AppDbContext

```csharp
public class AppDbContext : DbContext, IUnitOfWork
{
    private readonly IDomainEventDispatcher _dispatcher;
    private readonly ICurrentUserContext _currentUser;
    private readonly IDateTimeProvider _dateTime;

    public AppDbContext(
        DbContextOptions<AppDbContext> options,
        IDomainEventDispatcher dispatcher,
        ICurrentUserContext currentUser,
        IDateTimeProvider dateTime) : base(options)
    {
        _dispatcher = dispatcher;
        _currentUser = currentUser;
        _dateTime = dateTime;
    }

    public DbSet<Order> Orders => Set<Order>();
    public DbSet<Customer> Customers => Set<Customer>();

    protected override void OnModelCreating(ModelBuilder modelBuilder)
    {
        modelBuilder.ApplyConfigurationsFromAssembly(Assembly.GetExecutingAssembly());
        base.OnModelCreating(modelBuilder);
    }

    public override async Task<int> SaveChangesAsync(CancellationToken cancellationToken = default)
    {
        // Set audit fields
        foreach (var entry in ChangeTracker.Entries<AuditableEntity>())
        {
            if (entry.State == EntityState.Added)
                entry.Entity.SetCreated(_dateTime.UtcNow, _currentUser.UserId.ToString());

            if (entry.State == EntityState.Modified)
                entry.Entity.SetUpdated(_dateTime.UtcNow, _currentUser.UserId.ToString());
        }

        var result = await base.SaveChangesAsync(cancellationToken);

        // Dispatch domain events after successful save
        var aggregates = ChangeTracker.Entries<IAggregateRoot>()
            .Select(e => e.Entity as IHasDomainEvents)
            .Where(e => e?.DomainEvents.Any() == true)
            .ToList();

        foreach (var aggregate in aggregates)
        {
            await _dispatcher.DispatchAsync(aggregate!.DomainEvents, cancellationToken);
            aggregate.ClearDomainEvents();
        }

        return result;
    }
}
```

## Naming Conventions

| Type | Convention | Example |
| --- | --- | --- |
| DbContext | `AppDbContext` or project-specific | `ShopDbContext` |
| EF configuration class | EntityName + `Configuration` | `OrderConfiguration` |
| Repository implementation | AggregateName + `Repository` | `OrderRepository` |

---

## Important Notes

- Domain entities must have **no EF annotations** (`[Table]`, `[Column]`, `[Key]`). All mapping is done in `Configurations/` using the fluent API.
- Migrations are generated artifacts — do not edit them manually. If a migration needs adjusting, add a new migration.
- The `AppDbContext` implementing `IUnitOfWork` is a common pattern to avoid introducing a separate UoW wrapper. The context itself coordinates the transaction.
